部署工具

Git

使用 git 部署前，需要先下载包：hexo-deployer-git

修改配置文件 _config.yml ：

参数 描述 默认
repo 库（Repository）地址
branch 分支名称 gh-pages (GitHub) coding-pages (Coding.net) master (others)
message 自定义提交信息 Site updated: {{ now('YYYY-MM-DD HH:mm:ss') }})
token 可选的令牌值，用于认证 repo。用 $ 作为前缀从而从环境变量中读取令牌

远程仓库

Github

创建新仓库的名字为：<username>.github.io，在 setting > pages > Build and deployment > Source(Deploy from a branch) > branch 中设置分支为本地推送的分支 ，等待几分钟，访问<username>.github.io 即可

下载依赖包

hexo-renderer-stylus依赖下载hexo时存在则可以不用下载

简单配置

通过替换配置文件，_config.<theme>.yml 替换主题配置，如_config.butterfly.yml

一些坑

  1. _config.yml 文件中key：relative_link 必须设置为 false ，否则路径时将会以../的相对路径表示，某些情况将无法访问到
  2. 部署博客时，远程仓库的URL设置为 https|http 的格式时，将会出现 spawn failed 错误而无法部署，解决办法：修改URL为SSH格式
  3. 使用butterfly主题时，categories和tag页面出现404，解决办法：
    • hexo new page categories
    • hexo new page tags
    • 在生成文件夹的 index.md 中的元数据行添加，type: categories type: tags
      • 原理类似于指定脚本渲染本页面

1. 图书管理系统

描述

图书管理系统的功能是，普通用户可以对书籍进行查询（多种查询手段）、借阅、归还，系统管理员可以对书籍进行添加、修改信息等操作，并可以处理普通用户的借阅和归还请求。请根据上述功能叙述，提取这个系统的类，表示出每个类必要的属性和方法。

提示：查询手段在将来可能会增加，你的设计如何支持这种增加。

  • 类可复用
  • 扩展方便

好方案

不好的方案

  • 函数重复冗余：User类中的函数与Book中的函数重复。
  • 无法扩展：需要在类中添加方法不符合软件设计准则。
  • 无法复用：类中的方法针对于某一种特定的情况，无法用于其他设计中。

我的方案

我的方案

总结

软件设计准则：

千万不要向之前已经写好的类中添加属性和方法；若要添加则得通过新增类的方式增加新功能。

最好的设计是使用多态，让类在运行时确定。该特性可解决复用的问题

  • 添加属性会导致实例化的对象的大小增加
  • 添加方法必要的会添加属性，同样的也会让对象大小增加；在c++中可以通过函数表的方式遍历执行某个类的所有函数，当增加方法时，会导致函数表改变从而修改源程序。

2. 怪物减血设计

描述

一个游戏项目，有多种怪物，并且将来可能新增怪物。玩家攻击怪物时，怪物减多少血是一个综合多种因素的策略，例如要考虑玩家的攻击力和怪物的某些特性。新增怪物时要求不能更改玩家和怪物打仗的过程，否则就不能做到快速完成增加怪物的工作。请设计Monster类和Player类，表示出与攻击、减血相关的属性和方法，达到上述要求。

提示：应该仔细考虑方法的参数。

  • 怪物可拓展

好方案

我的方案

我的方案

总结

使用的软件设计技术：委托(entrust)

系统并不知道如何模拟真实世界中的打架的过程。但系统知道，玩家攻击怪物，怪物就根据机制会减血。于是系统省略打架这一过程，委托怪物自己减血。

委托：将某一操作交给别人去完成，而自己仅仅只是一次函数调用。

3.窗口改变形状

描述

在一个项目中，需要允许用户在运行时刻改变窗口的几何形状，例如矩形、圆形等，并且将来还会增加新的形状。在用户切换窗口形状的时候，程序不应该出现性能不足而导致的窗口闪烁或撕裂等异常显示。还有另一个功能，程序经常需要得知窗口当前的面积，这个过程必须快速高效。请设计Window类和Shape类，表示出必要的属性和方法以及两者的关系，达到上述目标。

好方案

不好的方案

  • 通过刷新的方式更新窗口形状，会导致窗口闪烁
  • 一个类对应一个形状，最开始打开的主窗口和进程绑定，如果删除该类的对象将会导致进程直接结束，若不删除则会导致之前不需要的对象仍占用内存。
  • 过多的继承将会导致混乱，难以维护

我的方案

我的方案

总结

软件设计：组合优于继承，多用组合少用继承

组合：通过属性引用其他类，可以获得其他类的方法和属性。

4.Timer类的单例模式

描述

几乎每个项目中都有多处使用定时器的地方，有些典型的项目甚至会同时有上百个定时器的客户对象存在。但定时器是一种操作系统资源，大量使用定时器将会对操作系统造成负担。所以我们的项目中希望不论客户对象有多少个，但定时器对象只能存在一个。请设计Timer类，使得无论程序员犯什么错都只有一个Timer对象存在。

好方案

总结

单例模式：整个系统中只能存在有一个实例

固定的格式：

  • 私有的构造方法
  • 静态的属性theOnlyInstance，以赋值
  • 静态的获得实例的方法getInstance()
